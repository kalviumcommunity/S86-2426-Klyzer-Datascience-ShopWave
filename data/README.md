# Data Folder

This folder contains all datasets used in the project.

## Structure

```
data/
├── raw/          # Original, immutable data
└── processed/    # Cleaned, transformed data
```

## Guidelines

### `raw/` folder
- **Purpose:** Store original, unmodified datasets
- **Rule:** NEVER edit files in this folder
- **Usage:** Read-only source data
- **Examples:** CSV downloads, JSON APIs, Excel files

### `processed/` folder
- **Purpose:** Store cleaned and transformed datasets
- **Rule:** Generated by scripts, not edited manually
- **Usage:** Input for analysis and modeling
- **Examples:** Cleaned CSVs, merged datasets, feature-engineered data

## Best Practices

✅ **DO:**
- Keep raw data untouched
- Document data sources
- Use clear, descriptive filenames
- Version processed datasets if needed

❌ **DON'T:**
- Modify raw data files
- Mix raw and processed data
- Store outputs here (use `outputs/` instead)
- Commit large files to git (use .gitignore)

## Example Files

```
data/
├── raw/
│   ├── sales_2024.csv          # Original sales data
│   ├── customers.json          # Customer information
│   └── products.xlsx           # Product catalog
└── processed/
    ├── sales_cleaned.csv       # After data cleaning
    ├── sales_with_features.csv # With engineered features
    └── merged_data.csv         # Combined datasets
```
